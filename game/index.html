警告

这只是个游戏，没什么大不了


好游戏的标准

  1. 激发创造力
  2. 具有挑战性
  3. 自由和空间
  4. 简单和统一
  5. 够得着的胜利

游戏的多面性

  1. 设计者看到的那面
  2. 使用者看到的那面

炉石传说

  1. 专注于某个角色
  2. 参考别人已经成熟的攻略
  3. 按比赛节奏进行出牌
  4. 仆从是比赛的主体

如何做事？

  1. 目的：想要达到的目标。目的改变了，其他也要相应改变；目的消失了，其他也无必要了。
  2. 设计：预先规划，制定图样。测试目的是否合理；设定实施步骤和范围。
  3. 实施：实际施行。在设计的框架内逐步达到目标。
  4. 工具：工作时所用的器具。更有效地尝试各种设计；更有效的实施。
  5. 方法：
    1. 读万卷书
    2. 行万里路
    3. 名家指路

做事方法

  1. 每天最多做三件事
  2. 以结果为导向衡量工作进度
  3. 养成良好的生活习惯

创始人

  1. 产品：尽早发布，迭代式开发
  2. 市场：和用户之间没有其他人（初期：用户但求精不求多）
  3. 团队：寻找优秀的人才
  4. 方向：面向未来（密切关注公司的发展）
  5. 处事：冷静，坚韧，主次分明（对当前任务有贯彻的执行力）
  6. 性格：工作务实，行动迅速
  7. 基础：找个起点作为支撑，向前开拓
  8. 勇气：在陌生的领域需要勇气进行决断，但要记住这里的危险，在最短的时间找到方法降低这个危险

合伙人

  1. 处得来
  2. 技能互补
  3. 趣味相投
  4. 坚韧不拔

团队

  1. 先建好队伍，再分配任务
  2. 保持团队成员在一起工作
  3. 模式化工作，而不依靠个人
  4. 建立短而规律的出货周期

时间

  1. 树立一个目标
  2. 把握住灵感
  3. 明确工作进度
  4. 利用“玩”的时间

展示与讨论

ppt只能用于展示；技术并不能带来新思想；讨论需要互动和好奇心


面向未来

  1. 工作：没有铁饭碗
  2. 教育：紧跟变化，守住不变
  3. 行业：消亡加剧，支柱产业长期缓慢发展
  4. 投资：依循生活方式的转变和国家的政策调控
  5. 生活：以预见的眼光审视未来，寻找当前与未来的接洽点
  6. 其他：社会化发展模式将成为主流，封闭式发展将萎缩或者消亡

并行

  1. 将独立事件分开处理
  2. 将复合事件分解为独立事件
  3. 加速必须是由需求引起
  4. 代码锁简单但容易引起锁竞争
  5. 数据锁通过分散降低了锁竞争，但热门条目仍存在激烈竞争

翻墙

这是一种长期的艺术行为


投资游戏

  1. 银行定期存储利率：4.75%
  2. 余额宝利率：4.0% ~ 6.0%
  3. 银行理财产品：5.0% ~ 6.0%
  4. 基金：用于选择投资方向
  5. 股票：选定三个行业进行实际操作（新能源汽车，医疗器械服务，核能概念）

简单开发原则

  1. 将所有材料尽可能集中在一起
    1. 需求分析
    2. 程序设计

      3. 功能设计

      UI设计

    3. 软件测试
    4. 软件部署
    5. 软件维护
  2. 将需求、设计和实现放置在一起
    1. 单元测试
    2. 功能测试
  3. 用最熟悉的方式对问题和方案进行描述
  4. 线上是线下的延续
  5. 函数多并不代表易维护

    6. 函数 + 说明 = 易维护

  6. 想一下，如果XX有某项功能就好了
